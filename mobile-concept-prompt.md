# LaundryOps Mobile Concept Prompt

Date: 2026-05-15

Purpose:
Generate the first polished mobile design concept board for LaundryOps.

Screens:

1. Home / Command Center
2. Machine Detail
3. Work Order Detail
4. AI Repair Assist

Prompt:

Create a polished mobile UI concept board for an Android-first subscription app named "LaundryOps".

Show four tall Android phone screens side by side on a clean light neutral canvas. Each phone should show a different screen of the same app, using one coherent design system.

Product context:
LaundryOps is a professional maintenance command center for laundromat owners and technicians. It helps users keep machines running, track work orders, diagnose repairs with AI, search manuals, and control maintenance costs.

Audience:
Laundromat owners, operators, and technicians. The app should feel serious, practical, trustworthy, and built for real field work.

Visual direction:
Modern field-service command center. Mostly light interface with a compact dark header area, clean white surfaces, status rails, strong machine numbers, tight but comfortable spacing, simple line icons, and clear bottom navigation. Use professional Android mobile design, not a web dashboard squeezed into a phone.

Palette:
Background #F6F8FA, surfaces #FFFFFF, header ink #102033, text #18212B, muted text #647282, border #DDE5EC, primary blue #1D5FDB, teal #009C9A, AI accent #6658D3, running green #15803D, warning amber #D97706, down red #DC2626, waiting purple #7C3AED.

Typography:
Use a clean Android-native feel similar to Roboto or Inter. Strong bold machine numbers. Clear screen titles. Compact labels. High readability.

Icon style:
Consistent simple line icons, 2px stroke, rounded caps, no emoji icons.

Composition:
One image, four full-height phone mockups side by side. All screens must be readable, polished, realistic, and store-screenshot quality. Use real app UI text, not decorative placeholder gibberish. Avoid tiny unreadable labels.

Constraints:
No emoji. No giant gradient cards. No horizontal overflow. No decorative bokeh/orbs. No marketing hero. No beige or purple-dominant theme. No generic placeholder dashboard. No nested cards inside cards. No fake browser chrome. Keep the app practical and professional.
